# Portfolio Map

## Purpose
This file defines the role of each portfolio section so the site stays focused and coherent.

## Main page families

### Home
Purpose:
- communicate identity quickly
- create first impression
- point visitors toward selected projects
- establish trust and tone

Expected blocks:
- personal headline / intro
- selected work preview
- credibility/worked-with strip or list
- short personal statement
- contact/social CTA

### Work
Purpose:
- provide a clear overview of projects
- let visitors quickly scan project quality and fit
- guide users into deeper case studies

Expected blocks:
- work list/grid
- project thumbnail
- title
- category / tags
- role / contribution
- optional short teaser

### Project detail / case study
Purpose:
- show thinking, process, and outcome
- demonstrate design/product/engineering depth
- prove decision-making, not just visuals

Expected blocks:
- project intro
- problem/context
- role and constraints
- process and key decisions
- outcome / learning
- optional links or prototype/media

### About
Purpose:
- make the person memorable
- explain motivation, taste, and working style
- reinforce trust

Expected blocks:
- portrait or personal image
- short story / philosophy
- experience highlights
- optional timeline or values

### Footer / contact
Purpose:
- make contact easy
- reinforce brand consistency
- provide social links

Expected blocks:
- email
- social links
- optional back-to-top link
- copyright / small signature

## Priority rule
If a design or motion choice conflicts with project clarity, project clarity wins.
